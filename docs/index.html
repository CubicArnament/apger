APGer Engine v2.0
Flexible Python 3.12+ build engine for Arch Linux environments

Обзор

APGer - это гибкий Python 3.12+ движок сборки для Arch Linux сред, предназначенный для автоматического создания пакетов в формате APGv2 с использованием GitHub Actions.

APGer автоматизирует процесс сборки пакетов из исходного кода, генерации строгих метаданных JSON, контрольных сумм файлов и создания архивов .apg с автоматическими GitHub Releases.

Основные возможности:

  • Поддержка различных систем сборки (Meson, CMake, Autotools, Cargo, Gradle, Python PEP 517)
  • Автоматическое определение последних версий пакетов
  • Безопасная подпись пакетов с использованием PGP
  • Генерация документации с помощью Doxygen
  • Интеграция с GitHub Actions для автоматической сборки

Структура проекта

  • repodata/: JSON файлы, определяющие пакеты
  • .ci/: Движок сборки и конфигурация
  • .github/workflows/: CI/CD рабочие процессы
  • docs/: Документация и генератор документации
  • dist/: Выходные пакеты

Процесс сборки

  1. Парсинг JSON рецептов из repodata/
  2. Загрузка исходного кода
  3. Сборка с использованием указанного шаблона
  4. Создание APG пакета
  5. Подпись пакета с использованием PGP
  6. Размещение пакета в репозитории

Документация API

Для подробной документации API смотрите сгенерированные HTML файлы в директории docs/html.

Чтобы сгенерировать документацию, выполните:

make html_docs

Для просмотра документации онлайн перейдите по ссылке: API Документация

Установка

Для установки APGer Engine выполните следующие шаги:

  1. Клонируйте репозиторий: git clone https://github.com/username/apger.git
  2. Установите зависимости: pip install -r requirements.txt
  3. Настройте конфигурацию в соответствии с вашими потребностями

Использование

Для запуска APGer Engine выполните:

cd .ci && python main.py

Или используйте Makefile:

make build

Конфигурация

Конфигурация проекта осуществляется через JSON файлы в директории repodata. Каждый файл описывает один пакет и включает:

  • Информацию о пакете (имя, версия, архитектура)
  • Источник исходного кода
  • Шаблон сборки (meson, cmake, autotools, cargo, gradle, python-pep517)
  • Зависимости сборки
  • Сценарии сборки и установки

Примеры

Примеры конфигурационных файлов находятся в директории repodata/.

Пример простого пакета:

{
  "package": {
    "name": "example-package",
    "version": "1.0.0",
    "architecture": "x86_64",
    "description": "Example package description",
    "maintainer": "Your Name ",
    "license": "GPL-3.0",
    "dependencies": ["dep1", "dep2 >= 1.0"]
  },
  "build": {
    "template": "meson",
    "dependencies": ["build-essential", "meson", "ninja"]
  }
}

Устранение неполадок

Если у вас возникли проблемы с APGer Engine, проверьте следующее:

  • Установлена ли Python 3.12 или выше
  • Доступны ли все зависимости сборки
  • Правильно ли настроены пути к исходникам
  • Доступен ли интернет для загрузки исходников
